Configuration

The configuration information below are specific to this implementation. Some items on the configuration is specific to the Harvester Manager, as well as to the JSONFactory component.

Configuration entries

Entries described below are from the main server-model configuration file. Some entries may not exist in the console configuration.

Configuration Entry Purpose Remarks
harvest.jdbc.user The DB user name.
harvest.jdbc.pw The DB password
harvest.jdbc.url The JDBC url
harvest.jdbc.Dataset.query The query to use when pull the Dataset records. The query can contain named parameters, see "sqlParam" section. The JDBC ResultSet must contain the field names matching the dataset template, but in flattened, dot-point-delimited form. One approach to achieve this is by creating a view that meets the naming requirements.
harvest.jdbc.Dataset.sqlParam.last_harvest_ts The named parameter indicating the last time the harvest query was executed. This parameter allows for incremental harvests. The value of this parameter is updated at runtime through a script that is launched as part of the JSONFactory execution.
activemq.url The ActiveMQ url where the data will be sent. This is typically where ReDBox is installed.
